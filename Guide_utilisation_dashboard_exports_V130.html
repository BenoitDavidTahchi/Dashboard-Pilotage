1. Charger les données

Sélectionner le projet concerné, puis charger les fichiers disponibles dans les zones d’import.

1 Exports ACC — Journaux des fichiers EXE & CHA CSV/Excel. Import indispensable.
2 Mappage WF — statut global, étape actuelle, nom du workflow, rôles et dates.
3 Référentiel — Fichier de codification et libellés associés.
4 Membres — Liste des membres de l'espace de projet ADCRM - remplacement des identifiants techniques par les noms des membres.
5 Historique WF — Extraction des revues - commentaires par round et notes de progression.
6 Annexe — convention ou guide, facultatif.
  Conseil : utiliser les extractions les plus récentes et vérifier la Date d’export ACC avant de poursuivre.

2. Générer ou actualiser le dashboard
GÉNÉRER LE DASHBOARD
À utiliser après le premier chargement. La solution fusionne les sources et construit les indicateurs, graphiques, arborescences et tableaux.
 
FUSIONNER ET ACTUALISER
À utiliser lorsqu’un ou plusieurs fichiers ont été remplacés ou complétés dans une session déjà ouverte.

Après génération, utiliser les filtres de colonnes, l’explorateur de dossiers, les indicateurs ou les graphiques interactifs.

3. Différence entre les boutons d’export
EXPORT EXCEL EXE
Périmètre complet EXE/BPE. Exporte les fichiers physiquement présents dans les dossiers reconnus de 03-EXE et les classe par onglets métiers. À utiliser pour le registre global EXE/BPE.
EXPORT EXCEL CHANTIER
Périmètre complet Chantier. Exporte les fichiers physiquement présents dans les dossiers reconnus de 07_Chantier, organisés selon les lots et sous-dossiers.
EXPORT FILTRÉ EXE/BPE + CHAN
Résultat exact des filtres actifs. Exporte uniquement les documents visibles qui se trouvent physiquement sous 03-EXE ou 07_Chantier. À utiliser pour une liste ciblée par statut, rôle bloquant, lot, échéance ou workflow.
EXPORT ACC ARBORESCENCE
Vue documentaire selon la structure ACC. Regroupe les documents selon leur chemin réel Autodesk Forma / ACC avec niveaux de dossiers et couleurs d’arborescence.
  Important : Seuls les fichiers physiquement présents dans les dossiers reconnus de l'arborecence sont exportés.

4. Structure commune des fichiers Excel
1 00-Page de garde — projet, dates, type d’export, source et consignes.
2 Onglets des listes documentaires uniquement — aucun onglet d’audit ou de statistique intermédiaire.

Les 22 colonnes sont conservées dans cet ordre :

1. Nom du fichier
2. Indice
3. Version ACC
4. Format
5. Lot
6. Description
7. Statut Révision (Fichier)
8. Commentaires du dernier round
9. Historique des commentaires
10. Notes progression WF
11. Lien Fichier

12. ID
13. Statut ACC
14. Statut Global (WFs)
15. Étape actuelle
16. Nom WF correspondant
17. Rôles Obligatoires
18. Rôles Bloquants
19. Approbateur Final
20. Date Création
21. Échéance (Due le)
22. Date Clôture

5. Bonnes pratiques
•Contrôler la date de l’extraction ACC avant de générer les tableaux.
•Pour un export ciblé, vérifier le nombre de lignes visibles avant l’export filtré.
•Utiliser les hyperliens pour ouvrir directement le fichier source dans Autodesk Forma / ACC.
•Une information WF vide signifie qu’aucune correspondance suffisamment fiable n’a été confirmée.
•Après remplacement d’une extraction, utiliser Fusionner et Actualiser.

Ce fonctionnement permet de distinguer clairement les exports complets, l’export ciblé par filtres et la vue hiérarchique ACC.
